# 함수

## 특징
1 함수의 목적 : 중복을 줄이기 위함  
2 한 번에 한 작업에 집중  
3 읽기 쉽고 어떤 동작인지 알 수 있게 네이밍  
  ```javascript
  showError() //에러를 보여줌
  getName() // 이름을 얻어옴
  createUserData() //유저데이터 생성
  checkLogin() // 로그인 여부 체크
  ```
<br>

## 매개변수 없는 함수
  ```javascript
  function showError() {
    alert('에레가 발생했습니다. 다시 시도해주세요.')
  }
  
  showError();
  ```
<br>

## 매개변수 있는 함수
  ```javascript
  function sayHello(name) {
    const msg = `Hello, ${name}`; 
    console.log(msg);
  }
  
  sayHello('Yueun'); //"Hello, Yueun"
  
  function sayHello(name) {
    let msg = 'Hello'; //msg는 함수 밖에서는 못 씀
    if(name){
      msg += ', ' + name;
    }
    console.log(msg);
  }
  
  sayHello(); // "Hello"
  sayHello('Yueun') // "Hello, Yueun"
  
  ```
<br>

## 함수 선언문 
: 어디서든 호출 가능 (표현식보다 자유로움)
  ```javascript
  sayHello(); // 함수식 위에 있어도 상관없다!
  
  function sayHello(){
    console.log('Hello');
  }
  ```
<br>

  ## 화살표 함수(arrow function)
  : 전통적인 함수표현(function)의 간편한 대안
  ```javascript
  let add = function(num1, num2){
	return num1 + num2;
  }

  let add = (num1, num2) => {
    return num1 + num2;
  }

  let add = (num1, num2) => num1 + num2 
  // return문이 한 줄이라면 괄호 생략 가능

  let sayHello = name => 'HEllo, ${name}';
  // 인수가 하나라면 괄호 생략 
  
  let showError = () => { 
  alert("error");
  }
  // 인수가 없으면 괄호 생략 불가능

  let add = (num1, num2) => { 
  const result = num1 + num2;
  return result;
  }
  //return 전에 여러 줄의 코드 있을 경우, 일반 괄호 사용 불가능
  ```
